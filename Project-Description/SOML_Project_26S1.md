# Class Project

The course project is an opportunity for you to experiment with the Machine Learning and optimisation techniques covered in the course (and beyond, if you want). You can choose one of three possible projects, one of which gives you the freedom to design your own project (see below for guidelines).

A quick description of the three projects:

- **Project 1: learn to optimise.** Many optimisation problems involve hyperparameters. This project explores the idea of "learning" good hyperparameters using machine learning approaches. → [[Project_1_Learn_to_Optimise]]
- **Project 2: hands-on Large Language Models (LLMs).** This is a practical project where you will learn the basics of implementing LLMs. → [[Project_2_Hands-on_LLMs]]
- **Project 3: design your own project.** (See section below.)

## Project Report

Your project report should be **7–10 pages** long (pages that contain figures or plots are counted as normal pages but pages that contain only references are not counted), using A4 page size and 10 point font. You are free to structure the report how you think the information is best presented, but we will give a suggested report structure below which also indicates what information the report should contain. You can choose to follow the structure as below or modify it if you believe that will be more suitable given the project and approach you have chosen. The report should be easy to read and understand for a reader with general knowledge of Machine Learning techniques but not necessarily the exact methods you have chosen for your project. While the overall minimum and maximum page length requirements are strict, the page length indications in the suggested report structure are only to give you a rough indication of how much detail is expected for each of the sections. While you are expected to include the information described below in your project report, you are free to also include further information and discussions if you think it helps the reader to better understand important aspects of your project.

- **Abstract** (200 – 300 words). Give an abstract that concisely summarizes your project report in a similar way as you typically find in published papers.
- **Introduction** (1 – 2 paragraphs). The introduction should engage the reader, point out how and why the project you conducted is interesting / important / useful and what potential impact the results from the project could have.
- **Problem description** (1 – 2 pages). Give a detailed description of the problem you are trying to solve in your project and what objective you are trying to achieve. Give a detailed description and discussion of the data set you used, if any. Point out if there are particular difficulties or obstacles that need to be overcome to achieve the objective of your project.
- **Techniques and methods used** (2 – 3 pages). Discuss the methods / approaches / techniques you have chosen to achieve the objective laid out in the section before. You should also discuss the main advantages and disadvantages the various methods have and how you have mitigated these problems.
- **Presentation and discussion of results** (3 – 4 pages including plots). Present the solutions and results you have found to tackle the problems you set out to in the project. Use plots for illustration where appropriate. Discuss in particular how the results of different approaches you may have chosen differ and give possible reasons for any similarities and differences you identify.
- **Conclusion** (1 – 2 pages). In your conclusion, you can very briefly summarize the problem and solution you presented. The main part of the conclusion, however, should be dedicated to discussing the limitations each of your approaches have and how you would attempt to mitigate them if you had more time for the project. You should also discuss other potential ways of extending/continuing the work you did in the project and what results/impacts could be expected from such extensions.

The project will be graded on the following criteria: *Technical Quality of the Project Work* as well as quality of the report, subdivided into *Abstract and Introduction, Technical Description of Problem and Approach Used, Presentation and Discussion of Results,* and *Conclusion*. For the report, technical accuracy, clear structure and a use of language that makes the report easy to read and understand are important considerations in grading.

**Note**: In terms of technical content, Project 1 has a much longer project description than Project 2 which explains the preparation steps in detail. This does *not* mean Project 2 is easier. Please note that Project 2 has a separate Jupyter Notebook file that you can access on Canvas. The Jupyter Notebook provides most of the code already, but it does *not* mean you will get full marks by just running all the code.

## Project 3: Design Your Own Project

You can (and are encouraged) to propose your own project. Possible projects could fall into one of the following categories.

- Application of Machine Learning methods for real-world (engineering) problems
- Application of Convex Optimization methods for real-world (engineering) problems
- Topics on the intersection of Machine Learning and Optimization (akin to Project 1)
- **Theory.** Theoretical study of a specific problem in learning theory (e.g. a formal study of unsupervised learning; theoretical characterization of the performance of neural networks; theory of model selection, etc.) or optimization (e.g. specific optimization technique for training neural networks; theory behind regularization; etc.).

If you choose this project, a proposal is to be submitted (by email) to the teaching team (see Important Dates) for approval. The proposal should be less than 2-page long (A4 page size), but should contain enough information of background, problem statement, data, possible approaches and references.

We will be inclined to mark your project higher if you choose to design your project.
